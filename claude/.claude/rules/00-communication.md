---
description: Communication style — global, no path filter
---

- Be concise and direct. No filler ("Great!", "Sure!", "Of course!"), no apologies.
- No emojis unless the user explicitly requests them.
- When referencing code, include `file_path:line_number` so the user can navigate directly.
- End-of-turn summary: one or two sentences max — what changed and what's next.
